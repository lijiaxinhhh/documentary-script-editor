# Design Context

本轮 UI 只参考三类约束：`taste-skill`、`impeccable.style` 和 Codex 内置 `Frontend App Builder skill`。

## Taste Rules

- 一个主视觉观点：本地纪录片剪辑工作台。
- 一个主色：温暖琥珀用于关键操作；青色只作为状态辅助，不能喧宾夺主。
- 暗色使用 off-black，文字使用 off-white，不用纯黑纯白造成廉价对比。
- 减少 AI 生成感：不堆大渐变、不堆发光、不堆泛用胶囊和空洞说明。
- 控件半径克制，工具界面优先可扫读、可重复操作。

## Impeccable Rules

- 产品上下文优先于装饰：每个模块要回答“用户下一步该做什么”。
- 页面不做空泛英雄区，不用 ghost-card 式堆叠来伪装完成度。
- 组件继承同一套令牌：背景、边框、按钮、状态、纸稿表面和故事版卡片必须统一。
- 不能为视觉好看牺牲交互路径：转写 Key、导入素材、载入样例、导出工程必须一直可见。

## Frontend App Builder Rules

- 第一屏必须是真实可用的产品表面。
- 桌面和移动端都要验证：无横向溢出、无裁切、无框架错误覆盖层。
- 不新增无关依赖；继续沿用 React/Vite、现有组件、现有测试。
- 每轮改动都要浏览器截图检查，并跑 `typecheck`、e2e 和 build。

## Current Visual Direction

- 顶部：低高度命令栏 + 轻量步骤条，不做大标签页。
- 主区：左素材，中纸稿逐字稿，右视频字幕。
- 开工区：命令条而不是宣传横幅，重点放在“导入视频”和“设置转写 Key”。
- 故事版：暗色画布 + 纸质卡片，像剪辑桌上的可移动结构卡。
