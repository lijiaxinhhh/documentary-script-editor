# 分享与部署方案

这个项目可以让别人使用，但要区分两层：

- 前端网站：可以部署成公开静态网站，别人直接用浏览器打开。
- 本机转写执行层：如果用户要“视频转文字”，仍然应该在用户自己的电脑上运行，避免素材上传到网站服务器。

## 方案 A：同一 Wi-Fi 临时试看

适合让旁边的人快速试用界面和导入/故事版/导出流程。

```bash
pnpm dev:lan
```

然后查看本机局域网 IP：

```bash
ipconfig getifaddr en0
```

同一 Wi-Fi 的用户打开：

```text
http://你的局域网IP:5174/
```

注意：这种方式只是把网站页面从你的电脑发给对方。对方选择的视频文件仍然只在对方浏览器里读取；但如果对方要转写，最好让对方在自己电脑上运行转写桥接服务，并在“视频转文字 / Key”面板里确认本机执行层地址是 `http://127.0.0.1:8787`。

## 方案 B：公开网站

适合让任何人打开网页使用基础剪辑功能。

```bash
pnpm build
```

把 `dist/` 目录部署到任意静态网站托管服务，例如 Vercel、Netlify、Cloudflare Pages、GitHub Pages 或自己的服务器。

如果部署到 GitHub Pages，使用：

```bash
pnpm build:pages
```

公开地址会是：

```text
https://lijiaxinhhh.github.io/documentary-script-editor/
```

公开网站上线后：

- 导入视频、导入 JSON、搜索逐字稿、故事版、反馈本地保存、导出文件都可以直接在浏览器里运行。
- 用户的素材不会上传到这个静态网站。
- API Key 会保存在用户自己的浏览器 localStorage 中。
- 需要转写时，用户需要在自己电脑上运行本项目的本机桥接服务：

```bash
pnpm bridge:transcription
```

网页里的“本机执行层地址”默认是：

```text
http://127.0.0.1:8787
```

如果用户改了端口或桥接服务运行在别的本机地址，可以在转写面板里修改。

## 方案 C：给别人完整本地包

适合保持“完全本地优先”的产品承诺。

把项目打包给用户后，用户在自己的电脑上运行：

```bash
pnpm install
pnpm dev
pnpm bridge:transcription
```

然后打开：

```text
http://127.0.0.1:5174/
```

这条路径最稳，也最符合当前 MVP 的本地隐私边界。下一轮可以把它封装成 Electron 或 Tauri 桌面应用，减少用户安装 Node/pnpm 的门槛。

## 当前推荐

第一阶段推荐：

1. 用 `pnpm build` 部署一个公开静态网站，方便大家体验界面、故事版和导出。
2. 同时准备一个“本机转写桥接服务”安装说明，让需要转写的人在自己电脑上运行。
3. 收集反馈后，再决定是继续做公开网站 + 本地桥，还是封装桌面软件。
